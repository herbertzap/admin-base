nosotros actFLUJO DE PROCESO DE SERVICIOS PARA NUEVOS CLIENTES
==================================================

DESCRIPCIÓN GENERAL
-------------------
Este documento describe el flujo completo de atención de solicitudes de servicios para nuevos clientes, desde la recepción inicial hasta el cierre comercial. El proceso involucra múltiples departamentos y roles con responsabilidades específicas en cada fase.

ROLES Y RESPONSABILIDADES
-------------------------
• Gerente Alianzas: Revisión y aprobación de propuestas comerciales
• Gerente Comercial: Aprobación final de todos los documentos del flujo B2B
• Responsable Implementación B2B: Revisión técnica y planificación de implementación
• Operador Operación: Ejecución de tareas operativas y validaciones
• Mesa de Ayuda: Validación inicial y soporte técnico
• Proceso: Coordinación general y documentación

FASES DEL PROCESO
=================

FASE 1: INICIO
--------------
Objetivo: Recepción y validación inicial de la solicitud de servicio

Actividades por Rol:

OPERADOR OPERACIÓN:
1. Inicio Operación
2. Recepción de Solicitud de Servicio - Nuevo Cliente
3. Confirmación Booking
4. Validar si la información es correcta para continuar con el flujo
   - Si NO: Fin de Proceso - Solicitud Rechazada
   - Si SÍ: Continuar
5. Generar CRO
6. Enviar CRO
7. Cargar Cliente Internacional en SD-CRM
8. Notificar OOB y envío de datos de contacto

MESA DE AYUDA:
1. Inicio Mesa de Ayuda
2. Validar Solicitud de Servicio - Nuevo Cliente
3. Validar si la información es correcta para continuar con el flujo
   - Si NO: Fin de Proceso - Solicitud Rechazada
   - Si SÍ: Continuar
4. Generar CRO
5. Enviar CRO

4. Enviar Propuesta Comercial

MESA DE AYUDA:

2. Revisar Propuesta Comercial
3. Propuesta Aprobada?

PUNTOS DE DECISIÓN Y VALIDACIÓN
===============================
En cada fase existen puntos de validación que pueden resultar en:
• Continuación del proceso (Si)
• Rechazo y fin del proceso (No)
• Retroalimentación para corrección (ciclos de revisión)

PRE-REQUISITOS IMPORTANTES
==========================
• IMPLEMENTACIÓN: El cliente debe tener un ambiente de pruebas para la integración
• FACTURACIÓN: El cliente debe tener un ambiente de producción para la integración
• COBRANZA: El cliente debe tener un ambiente de producción para la integración
• CIERRE COMERCIAL: El cliente debe tener un ambiente de producción para la integración

NOTA IMPORTANTE PARA FLUJO B2B
==============================
Para el flujo de B2B, el Gerente Comercial debe ser el único responsable de aprobar:
• Propuesta
• Contrato
• Orden de Servicio
• Plan de Implementación
• Confirmación de Implementación
• Factura
• Cobranza

CRITERIOS DE REVISIÓN
=====================
Según Cliente: Precio, Plazo, Servicio, SLA, etc.

DOCUMENTACIÓN REQUERIDA
=======================
Para cada fase de revisión se requiere la documentación correspondiente:
• Documentación de la Propuesta
• Documentación del Contrato
• Documentación de la Orden de Servicio
• Documentación del Plan de Implementación
• Documentación de la Confirmación de Implementación
• Documentación de la Factura
• Documentación de la Cobranza

EVENTOS TEMPORALES
==================
Existen eventos temporales asociados a las actividades iniciales de cada rol, indicando plazos o deadlines específicos para la ejecución de las tareas.

RESULTADOS ESPERADOS
====================
• Proceso exitoso: Cierre Comercial y Fin de Proceso
• Proceso rechazado: Fin de Proceso - Solicitud Rechazada (en cualquier punto de validación)

Este flujo asegura un control riguroso de calidad y aprobación en cada etapa del proceso de atención a nuevos clientes, garantizando que todos los requisitos técnicos, comerciales y operativos sean cumplidos antes de proceder a la siguiente fase.
